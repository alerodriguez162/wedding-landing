# Reglas ESLint sugeridas (opcionales)

Estas reglas ya están aplicadas en `.eslintrc.json`. Otras que puedes considerar:

## Calidad de código
- **`max-len`**: Limitar longitud de línea (ej. 100–120).
  ```json
  "max-len": ["warn", { "code": 120, "ignoreUrls": true, "ignoreStrings": true, "ignoreTemplateLiterals": true }]
  ```
- **`complexity`**: Limitar complejidad ciclomática.
  ```json
  "complexity": ["warn", { "max": 15 }]
  ```
- **`max-lines-per-function`**: Límite de líneas por función.
  ```json
  "max-lines-per-function": ["warn", { "max": 80, "skipBlankLines": true }]
  ```

## TypeScript
- **`@typescript-eslint/explicit-function-return-type`**: Exigir tipo de retorno en funciones (muy estricto).
- **`@typescript-eslint/strict-boolean-expressions`**: Evitar `if (x)` con tipos no booleanos.
- **`@typescript-eslint/no-floating-promises`**: Avisar si una promesa no se espera ni se captura.

## React
- **`react/hook-use-state`**: Preferir que el valor inicial de `useState` sea una función cuando sea costoso.
- **`react/no-array-index-key`**: Evitar usar el índice como `key` (warn).

## Accesibilidad (eslint-plugin-jsx-a11y, viene con Next)
- **`jsx-a11y/click-events-have-key-events`**: Si hay `onClick`, asegurar teclado.
- **`jsx-a11y/no-static-element-interactions`**: Evitar manejadores en elementos no interactivos sin rol.

## Nota sobre `react/jsx-no-bind`
En la config actual está en **warn** con **`allowArrowFunctions: true`** para no marcar todos los `onClick={() => ...}`. Si quieres ser más estricto, cambia a `allowArrowFunctions: false` y usa `useCallback` o manejadores nombrados en componentes que se re-rendericen mucho.
